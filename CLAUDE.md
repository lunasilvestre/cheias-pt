# cheias.pt — The Winter That Broke the Rivers

## Environment
Always use the project virtual environment at `.venv/`. Activate with `source .venv/bin/activate` before any pip install or python3 command. NEVER use --break-system-packages.

## What This Is

A **geo-narrative** about Portugal's January–February 2026 flood crisis — what happened,
why it happened, and what it means. Told through maps, satellite imagery, and
hydro-meteorological data.

Portfolio piece targeting Development Seed (Lisbon + DC). Public service artifact for
Portuguese citizens, journalists, and local officials.

**The crisis:** Storm cluster (Kristin → Leonardo → Marta) killed 11+, displaced thousands,
collapsed the A1 motorway, burst the Mondego levee, triggered €2.5B aid package across 69
municipalities. Tejo at highest since 1997. Sado at levels unseen since 1989. CEMS rapid
mapping activated (EMSR861, EMSR864).

  Create `v2/phase-N` off main. Merge only after all acceptance criteria pass.

- Run `git status` before starting work. If the repo isn't clean, fix it first.

## Data Architecture

**Total footprint:** 2.9 GB across 7,447 files. Full inventory in `prompts/creative-reports/data-catalogue.md`.

### What lives where

| Location | What | Managed by |
|----------|------|-----------|
| **Cloudflare R2** | All COGs (722 MB), served via titiler.cheias.pt | `scripts/sync-data.sh`, per-dir `sync.sh` |
| **`data/`** | Local copies of everything — COGs, GeoJSON, PMTiles, PNGs, frontend JSON | `.gitignore` excludes all binaries |
| **`data/frontend/`** | Pre-processed JSON for scroll narrative (3 MB) | Generated by `scripts/` |
| **`data/raster-frames/`** | Pre-rendered PNGs for crossfade animation (14 MB) | Generated by `scripts/generate-cog-frames.py` |
| **Git** | Only tracks: code, prompts, docs, configs, sync scripts, READMEs | `.gitignore` manages the boundary |

### R2 endpoints

| Service | URL |
|---------|-----|
| **titiler** | `https://titiler.cheias.pt/` |
| **R2 public** | `https://pub-abad2527698d4bbab82318691c9b07a1.r2.dev/cog/` |

### Data sync

```bash
# Sync all data from R2 to local
./scripts/sync-data.sh

# Sync specific directory
./data/cog/sync.sh          # pull COGs
./data/cog/sync.sh push     # push COGs to R2
```

**Rule:** If a script generates data into `data/`, the output is gitignored by extension.
If you create a new data type, add its extension to `.gitignore` before committing.
